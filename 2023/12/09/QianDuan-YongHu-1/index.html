安装Node.js和NPM

简单讲：

Node.js是一个Javascript运行环境(runtime environment)，不是一个js文件，实质是对Chrome V8引擎进行了封装。Node.js 是一个让 Javascript 运行在服务端的开发平台。

npm是包管理器，跟java里面的maven类似，npm 使 JavaScript 开发人员可以快速方便地共享软件包。因此Node.js 其实内置了npm，也就是说安装了Node.js也就安装了npm。

环境配置（可选）

由于Node.js默认将安装的各类包和缓存都存放在C盘，因此可以选择更改Node.js环境配置，从而更改安装位置。

找到刚才安装的目录，在安装目录下新建两个文件夹node_global和node_cache

创建完毕后，使用管理员身份打开cmd命令窗口（打开方法见下方Tips），输入

npm config set prefix “你的路径\node_global” （复制你刚刚创建的“node_global”文件夹路径）

例如：npm config set prefix "E:\ProgrammingLanguage\Node\node_global"

npm config set cache “你的路径\node_cache” （复制你刚刚创建的“node_cache”文件夹路径）

例如：npm config set cache "E:\ProgrammingLanguage\Node\node_cache"

设置 - 系统 - 系统信息 - 高级系统设置 - 环境变量

在系统变量中点击新建

变量名：NODE_PATH

变量值：E:\ProgrammingLanguage\Node\node_global\node_modules（Node.js安装位置中的node_global文件夹+node_modules）

此时node_global中应该多出了一个node_modules文件夹，如果没有可以自己手动创建。

编辑用户变量中的Path，将node_global提前到AppData\Roaming\npm之前，或者是将默认的 C 盘下 AppData\Roaming\npm 修改成node_global的路径。

在系统变量中选择Path点击编辑添加NODE_PATH，随后一直点击确定，直到完全退出。

修改node_global和node_cache的权限

为了方面安装，我修改了node_global和node_cache文件夹的权限。

鼠标右击node_global的文件夹，点击属性，再点击安全，再点击编辑，将权限都勾上，随即点击确定即可，node_cache步骤同理。
