一、常用语句

二、新建

创建一个新的 git 版本库。这个版本库的配置、存储等信息会被保存到git 文件夹中

三、配置

更改设置。可以是版本库的设置，也可以是系统的或全局的

四、状态

显示索引文件（也就是当前工作空间）和当前的头指针指向的提交的不同

五、信息

获取某些文件，某些分支，某次提交等 git 信息

六、添加

添加文件到当前工作空间中。如果你不使用 git add 将文件添加进去，那么这些文件也不会添加到之后的提交之中

七、删除

rm 和上面的 add 命令相反，从工作空间中去掉某个文件

八、分支

管理分支，可以通过下列命令对分支进行增删改查切换等

九、检出

将当前工作空间更新到索引所标识的或者某一特定的工作空间

十、远程同步

远程同步的远端分支

十一、撤销

十二、提交

commit 将当前索引的更改保存为一个新的提交，这个提交包括用户做出的更改与信息

十三、diff

显示当前工作空间和提交的不同

十四、grep

可以在版本库中快速查找

十五、log

显示这个版本库的所有提交

十六、merge

合并就是将外部的提交合并到自己的分支中

十七、mv

重命名或移动一个文件

十八、tag

十九、push

从远端版本库合并到当前分支

二十、ci

二十一、rebase（谨慎使用）

将一个分支上所有的提交历史都应用到另一个分支上不要在一个已经公开的远端分支上使用 rebase.
