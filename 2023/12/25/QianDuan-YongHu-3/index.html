vue实战-3-构建动态菜单和上传文件

发表于2023-12-25|更新于2025-04-08|前端vuevue实战
|浏览量:

框架构建

API：在API中统一管理相关的接口，进行前后端交流。

assets：页面所使用的图片等资料。

components：拥有构建组件，我将非大型页面都存放此处。

router：用于进行路由跳转等内容

stores：用于pinia仓库的定义与创建

style：用于定义全局样式

utils：用于存放全局使用的工具。

views：用于构建不同页面

utils

目前utils中只包含了request.ts文件，用于文件请求

页面布局搭建

在views/main中进行设计与布局

主题页面通过Element Plus的Container 布局容器为基础进行页面的设计。

Header:目前仅用来展示网站名称。

Aside:根据用户选择的模型进行动态菜单栏的构建。

Main:用于展示数据与内容。并根据用户点击不同的el-menu-item显示不同的页面。

由于目前的需求比较简单，因此复杂的动态菜单栏独立出去，剩下的Header和Main写在一起。构建完成后，使用router.push更改main中 显示的页面。

动态菜单栏

动态菜单栏的构建我分成了两个部分，第一个部分是菜单栏，第二个部分则是菜单按钮

Aside

我将Aside单独抽成了一个组件，命名为AsideBar。因为单击菜单栏时需要弹出对话框，并上传不同类型的数据。

Menu

由于菜单栏需要进行递归，因此在上半部分是没有子菜单的渲染为el-menu-item，而有子菜单的则渲染为el-sub-menu。

数据格式

useMenuStore

而所有的数据都是由后端传递过来的数据的JSON数据。并且存储在Pinia中。

api

type.ts

上传文件

由于需要通过点击el-menu-item才能弹出对话框，因此我将对话框DOM写在了Aside中，并将控制对话框是否显示的变量dialogTableVisible写在了useMenuStore中。

并且使用props进行组件间通信，确保UpLoadFile中可以获取必要的信息。

UpLoadFile

通过后端传递的文件上传类型确定上传的是哪种类型的文件。

UpLoadFilesStore.ts

为了更好地管理用户上传之后的数据，引入了pinia仓库。
