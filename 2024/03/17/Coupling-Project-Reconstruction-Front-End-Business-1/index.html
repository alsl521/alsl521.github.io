vue简介

Vue是一款用于构建用户界面的 JavaScript 框架。它基于标准 HTML、CSS 和 JavaScript 构建，并提供了一套声明式的、组件化的编程模型，帮助你高效地开发用户界面。

项目准备

项目清理与文件夹构建

由于创建Vue时，会自动创建部分无用的内容，因此需将项目重新整理。删除components和views中的所有内容。

然后新建如下文件夹

api：存放前后端交互的Url链接和数据格式。

assets：存放前端搭建页面时所需要的静态资源。

components：存放vue的组件。

router：存放vue的路由。详细内容可参考 Vue Router (vuejs.org)

styles：存放搭建前端页面时，可能需要的公共样式。

utils：存放前端可能需要的部分功能，例如封装的axios。

views：vue的视图组件。

App.vue：vue的根页面。

引入需要的插件

本项目使用Scss样式，所以需要安装相关依赖。

由于需要进行前后端通信，因此需要安装axios。

由于在使用Vue3的组合式 API 时，无法直接指定名称，并且本项目开发时，采用的是文件夹/index.vue的格式，所以引入vite-plugin-vue-setup-extend从而直接指定名称，帮助调试与开发。

还引入了element-plus，具体配置请看官网：快速开始 | Element Plus (element-plus.org)

其余的插件将在使用时一一介绍。

登陆页面

登陆页面中主要功能为登录，但当用户输入完用户名和密码之后，可以点击“记住密码”查看是否选择记住密码，如果选择了记住密码，则需要在“Cookie, SessionStorage 或LocalStorage”中存储用户名和密码，并在再次访问登陆页面时自动填充用户名和密码。

总体流程

  1. 除了用户跳转到登陆页面、注册页面和404页面外，首先判断用户浏览器中是否存储了Token，如果存储了Token，向后台查询并询问Token是否合理，如果合理的话则进入到主页面并根据本地存储的用户名和密码更新Token；如果不合理或不存在则跳转到Login页面。
  2. 在第一次进入Login页面时，输入用户名和密码。并可以选择是否记住密码，如果选择了记住密码，则将密码加密后存储在LocalStorage。
  3. 在后续进入Login页面时，如果LocalStorage里有用户信息时，则判断用户选择了记住密码，从而将用户信息自动存入表单里。如果用户取消勾选了记住密码，则会将LocalStorage中的信息消除。
